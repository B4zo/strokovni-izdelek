# Aplikacija Tehnični PreglediAplikacija Tehnični Pregledi je namenjena poenostavljanju vodenja pisarniških storitev v podjetjih, ki se okvarjajo s tehnični pregledi, registracijami, zavarovanji in homologiranjem vozil. Omogoča enostavno upravljanje dokumentov strank in vozili. Omogoča možnost spremljanja in spreminjanja podatkov o vozilih (tehnični pregledi, registracije, itd.) in sklepanja zavarovalnih polic. Vgrajena so tudi orodja za analizo in statistiko, ter zanesljivo poizvedovanje različnih podatkov.---## Načrtovane Funkcionalnosti### Uporabniki (Zaposleni)- Prijava in odjava uporabnikov,- Registracija novih uporabnikov, vključno z dodeljevanjem vlog.### Upravljanje s strankami in vozili- Dodajanje novih strank (fizične in pravne osebe),- Dodajanje novih vozil,- Pregled in urejanje podatkov o strankah in vozilih.### Upravljanje s storitvami in nalogami- Upravljanje tehničnih pregledov (dodajanje, urejanje, brisanje),- Upravljanje registracij vozil (dodajanje, urejanje, brisanje),- Upravljanje zavarovalnih polic (dodajanje, urejanje, brisanje).### Statistika in analiza podatkov- Analiza in statistika podatkov o vozilih in storitvah.---## Razlaga delovalnega modela aplikacijeAplikacija temelji na upravljanju strank in njihovih vozil ter na evidentiranju storitev, ki jih podjetje opravlja v okviru tehničnih pregledov, registracij in zavarovanj. Osnovna entiteta sistema je **stranka**. Vsaka stranka je lahko lastnik enega ali več **vozil**. Za posamezno vozilo sistem vodi evidenco pomembnih podatkov in povezanih postopkov, kot so:- homologacija,- tehnični pregledi,- zavarovalne police,- registracije.Ko stranka obišče podjetje, se zanjo ustvari **storitev**, ki predstavlja celoten obisk oziroma poslovni proces. Storitev je sestavljena iz ene ali več **nalog**, pri čemer vsaka naloga predstavlja konkretno opravljeno dejanje nad vozilom.Primer: ob enem obisku se lahko nad vozilom izvede **tehnični pregled**, sklene nova **zavarovalna polica** in opravi **registracija**. V tem primeru gre za **eno storitev**, ki vsebuje **tri ločene naloge**.---## Uporabljene TehnologijeCeloten sistem je zasnovan kot rešitev z dvema ločenima deloma, ki skupaj tvorita celovito aplikacijo za podporo pisarniškim postopkom v podjetju.### Arhitektura sistemaRazvoj aplikacije poteka v okolju Visual Studio 2026. Za vse dele aplikacije je uporabljen .NET 8 (8.0.24).Aplikacija je sestavljena iz dveh glavnih delov:- ASP.NET Core Web API in PostgreSQL baze,- Windows Forms aplikacije.#### Server - ASP.NET Core Web APIASP.NET API predstavlja osrednjo poslovno logiko sistema ter skrbi za:- obdelavo zahtev od namizne aplikacije,- validacijo podatkov,- dostop do podatkovne baze,- zagotavljanje enotnega in kontroliranega vmesnika za delo s podatki.Uporabljeni so različni NuGet paketi:- **Microsoft.EntityFrameworkCore**: za delo z bazo podatkov preko Entity Framework Core,- **Npgsql.EntityFrameworkCore.PostgreSQL**: za podporo PostgreSQL baze,- **Microsoft.AspNetCore.Authentication.JwtBearer**: za implementacijo avtentikacije z JWT žetoni,- **Swashbuckle.AspNetCore**: za generiranje Swagger dokumentacije in testiranje API-ja.- in drugi.#### Baza - PostgreSQLPodatki se shranjujejo v PostgreSQL bazi, ki je priljubljena odprtokodna relacijska baza podatkov. Baza vsebuje tabele za stranke, vozila, storitve, naloge in uporabnike, ter omogoča učinkovito shranjevanje in poizvedovanje podatkov. #### Odjemalec - Windows FormsWindows Forms aplikacija je namenjena uporabnikom (zaposlenim) za interakcijo s sistemom. Omogoča:- prijavo in odjavo uporabnikov,- upravljanje strank in vozil,- upravljanje storitev in nalog,- prikaz statistike in analize podatkov.
